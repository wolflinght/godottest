《坛说：江湖》战斗系统详细策划文档 (V1.3)
1. 核心概述
游戏类型：多人联机2D武侠RPG。
战斗表现：横版对峙线性站位（无九宫格，无阵法），极简UI（无集气条、无冗余Buff图标），核心体验由强文字播报驱动。
核心机制：服务器端隐藏时间轴（ATB）驱动，联机玩家与NPC共享同一进度条判定。兵刃与功法强绑定，拥有硬核的“变招”代价机制。
开发原则：重逻辑演算，轻量化表现。防外挂，一切判定在服务器执行。
2. 战斗属性与资源
2.1 基础属性转化
基础属性	转化战斗属性	作用说明
臂力	物理攻击 (ATK)、招架率	决定基础伤害与兵器格挡概率。
身法	集气速度 (SPD)、闪避率	决定隐藏集气条的增长速度。身法越高，出手频率越快。
根骨	气血上限 (HP)、内功防御	决定生存能力与对内功的抵抗力。
悟性	/	（战斗内无直接加成，主要用于系统外功法修炼速度）。
2.2 战斗资源
气血 (HP)：降至0则重伤退场。
内力 (MP)：主动资源。仅用于释放【绝招】与使用【吸气】回血。普通攻击不消耗内力。
3. 时间轴与联机判定机制 (引擎底层核心)
无论场上有多少名玩家和NPC，所有人都在同一个底层时间轴上进行排序判定，无需阵型。
3.1 隐藏时间轴 (Timeline)
服务器维护一个阈值为 1000 的进度条。
战斗开始后，每Tick（例如0.1秒），场上所有存活角色的进度 +该角色的集气速度(SPD)。
当某角色进度 >= 1000 时，服务器向该客户端发送 S2C_TurnStart（获得行动权），同时时间轴不停（其他角色的进度条仍在暗中积攒）。
3.2 玩家决策与超时机制
客户端收到 TurnStart 后，右下角弹出**【操作面板】**，并开始 10秒倒计时。
玩家在10秒内点击面板按钮发送指令，面板关闭，角色执行动作，该角色集气条清零重跑。
超时处理：若10秒倒计时结束未操作，系统自动向服务器发送**【攻击】指令（默认随机锁定**一名存活敌方），消耗本回合。
4. UI面板与操作指令集 (UI前端逻辑)
执行面板上的任意有效指令均消耗本回合。
第一排（战斗指令）：
[ 攻 击 ]：自动锁定敌方（随机锁定一名存活的敌方目标），按概率打出当前外功的随机招式。
[ 绝 招 ]：手动点击选择敌方目标，消耗内力，100%打出当前外功的大招。(若当前无功法，按钮置灰/隐藏)
[ 吸 气 ]：消耗内力，恢复自身气血。
第二排（战术指令）：
[ 换兵刃 ]：点击弹出背包武器列表。选择后执行**【智能换装】**，消耗本回合行动权。
[ 换功法 ]：点击弹出当前兵刃对应的已学武学列表。选择后完成同流派变招，消耗本回合行动权。
5. 兵器谱与功法匹配系统 (Smart Switch & Flailing)
所有兵器与外功均有底层标签（Tag）：拳脚(Unarmed)、刀法(Blade)、剑法(Sword)。
5.1 战前配置 (Out-of-Combat Loadout)
玩家需在系统面板中为每种兵刃类型设置**【默认功法】**（如：拳脚设为《太极拳》，刀法设为《八卦刀》）。若未学习该类武学，则槽位为空。
5.2 战内智能切换机制 (Smart Switch)
换兵刃联动：玩家点击[换兵刃]从【空手】换为【倚天剑】。系统检测到预设剑法为《太极剑》，自动在同一回合内完成“装备剑 + 切换太极剑”。
播报：{施法者}手腕一翻，抽出【倚天剑】，顺势摆出了《太极剑》的起手式！
同兵刃换功法：玩家拿着剑，点击[换功法]。列表智能过滤，只显示剑法供切换。
5.3 乱挥机制 (Unskilled Flailing)
触发条件唯一：玩家装备了一把武器，但他未学习过任何对应的功法（当前外功状态为“无”）。当他点击[攻击]时，必定触发乱挥。
惩罚数值：伤害强制变为原数值的 20%，命中率降低 50%。无法使用任何大招。
乱挥播报模板：
有兵器无功法：{施法者}根本不懂刀剑之术，只能凭蛮力胡乱挥舞手中的【{武器名}】，破绽百出，惹人发笑！
无兵器无功法：{施法者}毫无章法可言，像市井无赖一般朝{受击者}胡乱抓打过去！

7. 异常状态：点穴 (Stun)
战斗中唯一的负面硬控状态。剥离传统回合概念，完美契合时间轴。
机制原理：中招后，服务器停止该角色集气条的增长，并挂载一个“冻结时长”。
时长标准：点穴效果定义为 冻结 1000 点集气值的时间（即强行跳过一次出手机会）。
触发播报：{目标}被点中要穴，动弹不得！
解除播报：{目标}真气冲破穴道，恢复了行动。
8. 战斗判定与伤害公式 (程序基础计算)
命中率判定：
最终命中率 = 80%基础命中 + (攻击方命中附加 - 守方闪避率)
(上下限为5%~95%。未命中则播报：{目标}身形一闪，躲过了这招。)
招架判定 (仅命中后且守方持有武器时计算)：
招架概率 = 守方臂力带来的招架率
(招架成功伤害减半。播报：{目标}举起武器格挡，化解了部分力道。)
最终伤害计算：
基础伤害 = 攻击方ATK * 招式伤害系数
实际伤害 = (基础伤害 - 守方DEF) * 浮动RNG(0.9~1.1)
(若处于“乱挥”状态，实际伤害再乘以0.2。最低保底1点伤害)
